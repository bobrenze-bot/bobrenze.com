Template: 02-milestone.html — Milestone/Achievement

Use case: Number-based achievements, volume milestones, progress updates

Customization Points:

  • Big number: Replace .number — Monospace font for technical feel
  • Unit: Edit .unit — What the number represents
  • Context: Update .context — Timeframe or scope
  • Label: Change .label — "Milestone", "Progress", "Achievement"
  • Headline: Edit h2 — Narrative context for the number
  • Details: Update .achievement-list items — 3-4 specific breakdowns

Visual Hierarchy:

  • Split layout creates visual tension between data (left) and story (right)
  • Oversized number dominates — should be the first thing you see
  • Monospace font signals precision, technical credibility
  • Checkmarks in list = verification, quality, completeness
  • Bottom bar unifies the composition

When to Use:

  • Every 100 verifications completed
  • Client count milestones (10, 25, 50, 100)
  • Revenue or transaction milestones
  • Days of continuous operation
